# Electron+Vue3+TS 企业级研发模板需求文档

# 1. 文档概述

## 1.1 文档目的

本文档明确基于 Electron-Vite 脚手架开发的 Electron+Vue3+TS+ElementPlus 企业级研发模板的全部需求，涵盖技术栈、核心功能、架构设计、测试规范、文档要求等，作为项目开发、验收、维护及开源的核心依据，确保模板具备企业级可用性、可扩展性、安全性，同时满足开发效率与标准化要求。

## 1.2 项目定位

本项目是一款**企业级桌面应用研发模板**，基于前沿技术栈构建，内置完整的工程化规范、核心功能模块及测试体系，旨在降低 Electron 桌面应用的研发门槛，提供可直接复用、可快速扩展的基础架构，支持企业级应用快速落地，同时作为开源项目持续迭代，适配各类桌面端业务场景（如后台管理客户端、本地工具类应用、带服务端能力的桌面应用等）。

## 适用范围

本需求文档适用于模板开发团队、测试团队、维护团队，以及使用本模板进行二次开发的企业研发人员，作为开发实施、功能验收、文档编写、开源维护的统一标准。

# 2. 项目背景与目标

## 2.1 项目背景

当前 Electron 桌面应用研发过程中，存在架构不规范、技术栈零散、核心能力（如 IPC 通信、本地存储、服务端集成、AI 能力、云存储）需重复开发、测试体系缺失、文档不完善等问题，导致企业研发效率低下、应用稳定性不足、维护成本高。同时，市面上缺乏同时集成 Hono 服务端、AI 能力、多方案存储、完整测试及文档的企业级模板，因此需构建一款标准化、高性能、可复用的 Electron 研发模板，兼顾企业级生产需求与开源扩展性。

## 2.2 核心目标

- 技术标准化：基于 Electron-Vite+Vue3+TS+ElementPlus 构建标准化架构，配套完整的工程化规范（代码规范、目录规范、配置规范）。

- 功能完备性：集成 Hono 主进程服务端、AI 能力（Vercel-ai-sdk+OpenAI）、完整 IPC 链路、多方案本地存储（加密）、双平台云存储（工厂模式）。

- 可扩展性：模块解耦设计，支持业务模块快速扩展、技术栈平滑升级，适配不同企业的二次开发需求。

- 测试全覆盖：实现主进程、渲染层 Vitest 自动化测试，保障模板稳定性与可维护性。

- 文档完善性：配套 Rspress 生产级文档，涵盖安装、使用、扩展、API 说明等，降低使用门槛。

- 开源适配：代码规范、文档清晰、无闭源依赖，支持持续开源迭代，适配社区贡献与反馈。

# 4. 架构设计需求

## 4.1 整体架构

采用 Electron 标准多进程架构，严格区分主进程、渲染进程、预加载脚本，实现模块解耦，同时集成 Hono 服务端、AI 模块、存储模块，整体架构如下：

1. 主进程（main）：负责窗口管理、Hono 服务端、IPC 主端、AI 核心、云存储、本地存储、系统原生能力，是整个应用的核心枢纽。

2. 预加载脚本（preload）：负责 IPC 通信桥接，实现主进程与渲染进程的安全通信，禁止直接暴露 Node API，确保应用安全。

3. 渲染进程（renderer）：负责 Vue3 前端界面渲染、ElementPlus 组件展示、Vercel-ai-sdk-ui 集成、前端请求（调用主进程 Hono API）、用户交互。

4. 共享模块（shared）：负责主进程与渲染进程共享的 TypeScript 类型、工具函数，确保类型统一。

5. 测试模块（test）：负责主进程、渲染进程的 Vitest 测试用例编写与执行。

6. 文档模块（docs）：基于 Rspress 构建的生产级文档，涵盖模板使用、扩展、API 说明等。

# 5. 核心功能需求

## 5.1 工程化基础功能

### 5.1.1 项目初始化

- 基于 Electron-Vite 初始化项目，自动生成标准目录结构、配置文件（tsconfig、electron-vite.config.ts 等）。

- 集成 pnpm 包管理，配置完整的脚本（dev、build、test、docs:dev、docs:build 等）。

- 内置 ESLint + Prettier + Husky + lint-staged，实现代码规范校验、自动格式化、提交规范强制。

### 5.1.2 多环境配置

- 支持开发、测试、生产三环境，主进程与渲染进程均可访问环境变量。

- 敏感配置（如 OpenAI API Key、云存储密钥）不提交 Git，通过环境变量或本地配置文件加载，确保安全。

## 5.2 主进程核心功能

### 5.2.1 Hono 服务端集成

- 在主进程内置 Hono 服务端，支持本地（127.0.0.1）和局域网（0.0.0.0）访问切换。

- 封装完整的路由、中间件（CORS、日志、错误处理），提供统一响应格式。

- 实现基础 API 示例（健康检查、测试接口、存储操作接口等），支持请求参数校验。

- 集成静态文件服务，支持本地文件预览、上传下载。

### 5.2.2 AI 能力集成

- 主进程集成 Vercel-ai-sdk + OpenAI API，封装 AI 调用工具（对话、生成等），支持 API Key 配置。

- 实现 AI 调用异常处理、超时重试，确保稳定性。

- 支持 AI 能力开关，可根据需求启用/禁用，避免不必要的资源占用。

### 5.2.3 窗口管理

- 实现主窗口创建、销毁、隐藏、显示，支持无边框窗口（可拖拽、可伸缩）。

- 支持窗口大小、位置记忆，窗口置顶、全屏、最小化、最大化等基础操作。

- 支持多窗口管理（主窗口、设置窗口等），实现窗口间 IPC 通信。

### 5.2.4 存储模块集成

#### 5.2.4.1 本地存储（多方案 + 加密）

- electron-store：用于简单配置存储（如窗口配置、AI 开关、API Key 等），实现加密存储。

- lowdb：用于 JSON 文档存储（如用户数据、日志记录等），支持加密读写。

- node-sqlite：用于关系型数据存储（如复杂业务数据），实现数据库加密、表结构初始化。

- 封装统一的本地存储工具类，提供统一的调用接口，支持多方案切换。

#### 5.2.4.2 云存储集成（工厂模式）

- 采用工厂模式封装阿里云 OSS、腾讯云 COS SDK，实现统一的云存储接口（上传、下载、删除、查询等）。

- 支持云存储平台切换，只需修改配置即可切换阿里云/腾讯云，无需修改业务代码。

- 实现云存储密钥配置、异常处理、上传下载进度反馈。

### 5.2.5 日志系统

- 实现主进程、渲染进程双端日志记录，支持日志分级（info、warn、error、debug）。

- 日志本地存储，支持日志滚动、日志清理，便于问题排查。

## 5.3 预加载脚本功能

- 启用 contextIsolation 上下文隔离、sandbox 沙箱模式，确保应用安全。

- 封装 IPC 通信桥接，暴露安全的通信方法，禁止直接暴露 Node API。

- 实现 IPC 通信类型安全，与主进程、渲染进程共享类型定义，避免通信异常。

## 5.4 渲染层核心功能

### 5.4.1 基础前端架构

- 基于 Vue3 + TS 构建渲染层，集成 ElementPlus 组件库，实现企业级界面风格。

- 集成 Vue Router，实现路由管理（示例页面路由、404 路由等）。

- 集成 Pinia 状态管理，实现全局状态共享，支持状态持久化。

### 5.4.2 AI 交互 UI 集成

- 集成 vercel-ai-sdk-ui，实现 AI 对话、生成等交互界面，与主进程 AI 模块联动。

- 支持 AI 输入框、结果展示、加载状态、异常提示等交互体验。

### 5.4.3 前端请求封装

- 封装 axios，实现渲染层请求主进程 Hono API 服务，支持请求/响应拦截、超时处理、错误统一处理。

- 实现 API 调用类型安全，与 Hono 服务端接口类型统一。

### 5.4.4 示例页面
- 登录注册示例页面; 注册方式三种：普通邮箱验证码;普通账密;集成supabase验证;

- Dashboard 页面：展示模板核心功能入口、系统信息。

- IPC 测试页面：测试主进程与渲染进程 IPC 通信，展示通信效果。

- API 测试页面：测试 Hono 服务端接口，支持请求参数输入、响应展示。

- AI 测试页面：测试 AI 交互功能，展示 OpenAI 调用效果。

- 存储测试页面：测试本地存储（多方案）、云存储上传下载功能。

- 基于ElementPlus组件封装的通用常用业务组件使用展示;

- 本地调用Docker能力展示

- 一个完整的教师实体CRUD界面;

- 大屏界面展示;
## 5.5 完整 IPC 调用链路

- 实现主进程 ↔ 预加载脚本 ↔ 渲染进程的完整 IPC 通信链路，支持双向通信。

- IPC 通信类型安全，所有通信事件、参数均有 TS 类型定义，避免类型错误。

- 封装 IPC 调用工具（主进程、渲染层），支持异步调用、超时处理、事件监听/取消监听。

- 实现 IPC 通信白名单机制，禁止非法通信，提升应用安全。

- 示例：渲染层通过 IPC 调用主进程 AI 模块、存储模块、系统能力，主进程通过 IPC 向渲染层推送消息。

## 5.6 打包功能

- 基于 electron-builder 配置多平台打包（Windows：exe/msi；macOS：dmg；Linux：deb/rpm）。

- 实现打包后代码混淆、加密，保护源码安全。

- 配置打包图标、安装界面、版本信息，适配企业级发布需求。

- 支持离线打包，确保无网络环境下可正常安装使用。

# 9. 性能与安全要求

## 9.1 性能要求
